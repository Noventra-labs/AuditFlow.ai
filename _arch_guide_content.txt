[12] Project Overview
[13] FinancePilot is a production-grade multi-agent AI system that automates the full financial operations lifecycle for SE Asian SMEs: invoice ingestion, bank reconciliation, tax compliance, cash flow forecasting, and board reporting. It runs entirely on Google Cloud, integrates four MCP servers, and produces real auditable financial outputs — not just text summaries.
[14] 
[15] APAC IMPACT: SE Asia has 70M+ SMEs, most of which run their finances in spreadsheets with zero automation. FinancePilot directly addresses a $12B+ addressable market with a working, demo-able AI system.

[28] 02  SYSTEM ARCHITECTURE
[29] Google Cloud infrastructure, all 8 layers, service topology
[30] Architecture Philosophy
[31] FinancePilot is built on the principle that financial data is sacred. Every agent action is immutably logged to Supabase. No state lives in agent processes — all state lives in Redis (ephemeral) or Supabase (permanent). Every external call goes through Secret Manager. This architecture is both technically impressive and compliance-ready.
[32] 
[33] 8-Layer Architecture Overview
[34] Layer 1 — Client Interfaces
[35] The system exposes four interfaces: a Next.js financial dashboard with live P&L charts, an invoice upload portal with drag-and-drop PDF ingestion, a REST API for ERP integrations, and a WebSocket feed for real-time agent activity display during demos.
[36] Finance Dashboard: live charts (P&L, cash flow, AR aging), invoice list, reconciliation status, tax calendar
[37] Invoice Upload: drag-and-drop PDF/image upload, preview before processing, batch upload support
[38] REST API: POST /invoices, GET /ledger, GET /forecast, GET /reports — for ERP system integration
[39] WebSocket feed: real-time agent activity — what each agent is doing right now — excellent for demo
[40] 
[41] Layer 2 — API Gateway & Security
[42] 
[43] Layer 3 — Finance Orchestrator
[44] The Finance Orchestrator is a FastAPI application powered by Claude with financial domain expertise baked into its system prompt. It understands accounting concepts, SE Asian tax jurisdictions (GST Singapore, PPN Indonesia, VAT Thailand/Philippines), and common SME financial workflows.
[45] On each request, the orchestrator runs a two-phase Claude call:
[46] Financial Intent Analysis: identifies what financial operation is needed (invoice processing / reconciliation / tax / forecast / report)
[47] Task Decomposition: breaks the intent into subtasks mapped to specific finance agents
[48] Parallel Dispatch: publishes all subtasks to Pub/Sub topics simultaneously — agents run in parallel
[49] Result Synthesis: after collection, calls Claude again to produce a clean financial summary with action items
[50] 
[51] CRITICAL DIFFERENTIATOR: The orchestrator never hardcodes financial rules. It calls Claude with live context from Supabase + web search results to reason about current tax rates and regulations — making it genuinely adaptive to regulatory changes.
[52] 
[53] Layer 4 — Finance Sub-Agent Services
[54] 
[55] Layer 5 — Cloud Pub/Sub Message Bus
[56] All inter-agent communication flows through Cloud Pub/Sub. This means agents never directly call each other, have no awareness of each other's existence, and can fail and restart independently without affecting the rest of the system.
[57] finance.invoice: triggers Invoice Parser Agent when new invoice detected
[58] finance.reconcile: triggers Reconciliation Agent with invoice ID and bank statement window
[59] finance.tax: triggers Tax Compliance Agent with transaction details and jurisdiction
[60] finance.forecast: triggers Forecast Agent to update cash flow model with new data
[61] finance.report: triggers Report Agent to generate and email periodic summaries
[62] finance.alert: triggers Alert Agent for deadline notifications and anomaly warnings
[63] finance.results: all agents publish results here; Orchestrator subscribes to aggregate
[64] finance.dead-letter: failed messages after 5 retries routed here for inspection and replay

[71] 03  AGENT ARCHITECTURE

[73] Finance Orchestrator — Full Specification
[74] The orchestrator is a FastAPI service with Claude configured with a financial domain system prompt. It is the only entry point into the agent network — all user requests and all automated triggers (Gmail push, Cloud Scheduler) hit the orchestrator first.
[75] 
[76] Orchestrator System Prompt
[77] You are FinancePilot Orchestrator, a financial AI coordinator for SE Asian SMEs. You have deep knowledge of bookkeeping, GST (Singapore), PPN (Indonesia), VAT (Thailand/Philippines), cash flow management, and invoice processing. Your ONLY job is to analyze financial requests and decompose them into subtasks for specialized agents. Available agents: invoice_parser_agent, reconciliation_agent, tax_compliance_agent, forecast_agent, report_agent, alert_agent. Never execute financial operations yourself. Always return a JSON task plan with agent assignments, required data, and priority order.

[80] # orchestrator/main.py — FinancePilot Primary Orchestrator
[81] from fastapi import FastAPI, Depends, BackgroundTasks
[82] from google import genai
[83] from google.cloud import pubsub_v1
[84] import asyncio, json

[86] app = FastAPI(title="FinancePilot Orchestrator")
[87] client = genai.Client()

[89] # Core orchestration: decompose → dispatch → collect → synthesize
[90] async def process_financial_request(
[91]     message: str, company_id: str, session_id: str
[92] ) -> dict:
[93]     # 1. Analyze financial intent
[94]     plan = await decompose_financial_task(message, company_id)
[95] 
[96]     # 2. Dispatch all subtasks in parallel via Pub/Sub
[97]     dispatches = [
[98]         publish_to_agent(subtask, session_id)
[99]         for subtask in plan["subtasks"]
[100]     ]
[101]     await asyncio.gather(*dispatches)
[102] 
[103]     # 3. Collect results (Redis-based, timeout=45s for finance ops)
[104]     results = await collect_agent_results(
[105]         session_id, expected_count=len(plan["subtasks"])
[106]     )
[107] 
[108]     # 4. Synthesize financial summary with Claude
[109]     return await synthesize_financial_response(message, results)
