The aesthetic direction, principles and rationale  
The Design Direction — Luxury Minimal Dark  
FinancePilot's frontend is designed around a single powerful aesthetic: what would Bloomberg 
Terminal look like if Stripe redesigned it? The answer is a dark -first, data -dense, but breathable 
interface where every element earns its place. Financial profess ionals trust tools that look serious, 
precise, and capable — not playful or experimental.  
 
Core aesthetic direction: Luxury Minimal Dark. Dark carbon background (#0D1117), crisp green accent 
as the single dominant colour, Instrument Serif for hero numbers to add warmth, DM Mono for all 
financial data, generous whitespace within a dense informati on architecture.

Global Layout Architecture  
The global layout is a three -column structure that remains fixed across all tabs. Only the main 
content area changes.  
• Column 1 — Icon Navigation (52px fixed): vertical icon strip. Logo at top. 7 tab icons. Settings and 
profile at bottom. Tooltip labels appear on hover. No text labels — icons only, keeping the nav 
ultra-compact.  
• Column 2 — Context Panel (240px fixed): company name and logo, fiscal year selector, base 
currency badge, active account type (GST Registered / Non -GST). Below: sub -navigation for the 
active tab. Collapsible on narrow screens.  
• Column 3 — Main Content (flexible): changes per tab. Has its own internal header (breadcrumb + 
primary actions), content area, and optionally a right detail drawer (360px slide -in). 
• Global Header Bar (48px, spans columns 2+3): search ( ⌘K command palette), notifications bell 
with badge count, agent status indicators (6 coloured dots), dark/light mode toggle, user avatar 
menu.  
 
IMPORTANT: The layout never uses horizontal scrolling. All tables are either scrollable within their 
container or use responsive column hiding. On screens <1280px, column 2 collapses to a hover -
revealed panel. Mobile (<768px) uses a bottom tab bar replacin g the left sidebar entirely.

Component Specifications  
Cash Flow Chart (Recharts AreaChart)  
Three datasets: income (fill: rgba(22,163,74,.2), stroke: #22C55E), expenses (fill: 
rgba(220,38,38,.1), stroke: #ef4444 dashed), net (fill: none, stroke: #fff). CustomTooltip component 
shows all three values formatted with DM Mono. Responsive container — fills 100% of its parent 
width. No visible axes on mobile.  
Agent Activity Feed  
WebSocket connection to /ws/agent -events. Each event: { timestamp, agent_name, action, 
tool_called, result, confidence }. Rendered as a scrolling list of Event components. Color coding: 
green for success, amber for tool calls, red for errors. Pause/resume toggle stores events in a 
buffer when paused.  
KPI Cards  
Six MetricCard components in a CSS grid: grid -template -columns: repeat(6, 1fr). Each card fetches 
its value from a shared DashboardContext that subscribes to Supabase Realtime. On value 
update: number animates using a count -up library over 600ms.  
 
UX Principles for this Tab  
• The user should never need to click more than once to understand the current financial health of 
the company.  
• Data freshness indicators: each KPI shows "Updated 2 min ago" in muted text. If data is >15 
minutes old, show a stale indicator.  
• Empty states: if company is new, show a welcome flow with step -by-step setup guide instead of 
empty charts.  
• Colour accessibility: all status colours pass WCAG AA contrast ratio on the dark background.  
• Keyboard navigation: Tab key cycles through KPI cards, then chart period buttons, then action 
buttons.

Match invoices to bank transactions — auto-reconciliation with confidence scoring, manual

section below the split panels, collapsed by default. The top summary strip uses amber accents 
throughout.  
Surgical precision tool aesthetic — like a trade blotter on a trading desk. Two columns, clear 
connection between matched items, colour -coded confidence scores.
